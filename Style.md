# 后勤维修综合管理系统 交互与视觉设计规范 (V1.0)

## 一、设计原则

1. **简洁高效**：去除一切不必要的视觉装饰，聚焦核心任务路径，降低认知负荷。
2. **信息突出**：通过对比明确的色彩、字号、间距建立清晰的信息层级，关键操作一眼可辨。
3. **视觉舒适**：采用柔和中性色系，圆角统一，无阴影设计，长时间使用不疲劳。
4. **一致性**：所有终端（管理端、用户小程序端）遵循相同的设计语言和组件规范。
5. **可访问性**：确保文字对比度符合 WCAG AA 标准，触控区域足够大。

## 二、色彩系统

### 2.1 主色调
- **品牌青绿**：`#0f766e` —— 用于主按钮、选中态、关键操作、强调文字
- **浅色变体**：`#38a89d`（悬停态）、`#5dc4b6`、`#a7e0d8`、`#e0f5f2`（浅底）

### 2.2 中性色
- **应用背景**：`#f5f6f8` —— 页面底色
- **卡片/面板背景**：`#ffffff`
- **悬浮背景**：`#f8fafc`
- **激活态背景**：`#e8f4f2`
- **边框**：
    - 细边框 `#f0f2f5`
    - 常规边框 `#edeff2`
    - 加强边框 `#d0d5dd`
- **文字**：
    - 主要文字 `#1e293b`
    - 次要文字 `#64748b`
    - 辅助/禁用文字 `#94a3b8`

### 2.3 功能色
- **成功**：`#10b981`（背景 `#d1fae5`）
- **警告**：`#f59e0b`（背景 `#fef3c7`）
- **危险/错误**：`#ef4444`（背景 `#fee2e2`）
- **信息**：`#3b82f6`（背景 `#dbeafe`）

### 2.4 色彩应用规则
- 按钮：主要操作用品牌色，危险操作用红色，禁用态灰度。
- 状态标签：对应功能色浅底配深色文字。
- 表格表头背景 `#f7f8fa`，文字 `#606266`。
- 所有色彩均不使用渐变，保持扁平。

## 三、字体与排版

### 3.1 字体家族
- **中文**：`"PingFang SC", "Microsoft YaHei", sans-serif`
- **英文/数字**：`"Inter", -apple-system, BlinkMacSystemFont, sans-serif`

### 3.2 字号层级 (基于 375px 屏幕宽度，使用 rpx 单位)
| 层级 | 字号 | 用途 |
|:---|:---|:---|
| H1 | 38rpx / 19px | 页面主标题 |
| H2 | 32rpx / 16px | 卡片标题、分段标题 |
| H3 | 28rpx / 14px | 列表项标题、弹窗标题 |
| 正文 | 28rpx / 14px | 表单内容、描述文字、表格内容 |
| 辅助文字 | 24rpx / 12px | 时间戳、标签、次要描述 |
| 最小文字 | 22rpx / 11px | 角标、极辅助信息 |

### 3.3 行高与字重
- 正文行高：1.5 ~ 1.6
- 标题行高：1.3
- 字重：标题 `600` ~ `700`，正文 `400` ~ `500`，强调数字 `700` ~ `800`

---

## 四、间距与圆角

### 4.1 基础间距单位
- **基础网格**：4rpx
- **常用间距值**：8, 12, 16, 20, 24, 28, 32, 40, 48 rpx

### 4.2 组件内间距
- 卡片内边距：24rpx ~ 32rpx
- 列表项间距：18rpx ~ 24rpx
- 表单组件间距：20rpx ~ 24rpx
- 按钮与周围元素间距：16rpx ~ 24rpx

### 4.3 圆角规范
- **小圆角**：10rpx —— 按钮、输入框、标签
- **中圆角**：16rpx —— 卡片、面板、对话框
- **大圆角**：24rpx ~ 32rpx —— 特殊卡片、弹窗
- **全圆角**：999rpx —— 胶囊按钮、搜索框、标签

---

## 五、图标与组件规范

### 5.1 图标风格
- 统一使用线性图标，描边宽度 2px。
- 常用操作图标：`DataAnalysis`, `List`, `User`, `Goods`, `OfficeBuilding`, `Collection`, `Bell` 等（Element Plus 图标库）。
- 移动端可使用 emoji 作为简单图标替代，尺寸 48rpx。

### 5.2 按钮
- 高度：80rpx (小程序) / 36px (Web)
- 圆角：48rpx (小程序) / 10px (Web)
- 状态：默认、悬停（变深）、按下（透明度）、禁用（灰度+降低不透明度）
- 主要按钮填充品牌色白字，次要按钮浅底深字加边框

### 5.3 输入框
- 背景 `#f8fafc`，聚焦时边框变品牌色，外围出现 3px 浅色光晕（品牌色最浅变体）
- 圆角 22rpx（小程序） / 10px（Web）

### 5.4 标签
- 圆角 6px，高 24px，内边距 0 8px
- 色彩使用功能色浅底 + 深文字

### 5.5 表格
- 表头背景 `#f7f8fa`，文字 `#606266`，字号 13px，加粗
- 行悬停背景 `#f2f7ff`
- 行高 48px，行内间距 14px
- 无竖向分割线，横向分割线色 `#f0f2f5`

## 六、布局与网格

### 6.1 管理端布局
- **侧边栏**：固定宽度 240px，背景纯白，右侧细边框。菜单项圆角 12px，内缩 8px 边距。
- **主内容区**：左侧 margin-left: 240px，顶部 Header 56px，内容区 padding 28px，背景 `#f5f6f8`。
- **栅格**：使用 Element Plus 24 栅格，统计卡片区常用 `el-col :span="4"` 每行 6 个指标卡。
- **图表区域**：高度统一 340px ~ 420px，卡片内边距 20px。

### 6.2 用户端小程序布局
- **页面背景**：`#f5f6f8`
- **卡片**：背景白，边框 `#f0f2f5`，圆角 24rpx，内边距 28rpx，卡片间距 24rpx。
- **栅格**：常用双列网格 `grid-template-columns: repeat(2, 1fr)`，列间距 30rpx。
- **安全区**：底部留白 `padding-bottom: env(safe-area-inset-bottom)`。

## 七、交互状态与反馈

### 7.1 可点击元素
- 按钮、列表项、卡片：悬停时背景轻微变色（如 `#f8fafc`），按下时透明度变化或轻微缩放（0.96）。
- 链接文字：悬停时颜色加深。
- 禁用态：整体灰度，`opacity: 0.5` 或灰色背景。

### 7.2 加载状态
- 骨架屏：列表或卡片使用 Element Plus `v-loading` 或骨架屏组件，显示灰色占位块。
- 按钮加载：`loading` 属性开启时显示旋转图标，禁用点击。

### 7.3 空状态
- 统一使用 `EmptyState` 组件，包含插图和提示文字，可选操作按钮。

### 7.4 提示反馈
- 成功/失败操作使用顶部居中 Toast，持续时间 2s。
- 确认操作使用对话框 `ElMessageBox.confirm`，标题明确，按钮文案清晰。

### 7.5 列表与滚动
- 长列表使用 `scroll-view` 或原生滚动，配合 `refresher` 下拉刷新和 `scrolltolower` 触底加载。
- 管理端表格使用分页器，页码大小可切换。

## 八、关键页面设计规范

### 8.1 管理端 — 数据看板
- 顶部指标卡：一行 6 个，高度一致，数字大且加粗（28px），标签小号灰色（13px）。
- 图表区域：两行各两个，卡片带标题，图表内部无边框。
- 整体视觉焦点在数字和趋势线上，减少装饰元素。

### 8.2 管理端 — 工单列表
- 搜索栏胶囊式表单，背景白色带边框，与表格分离。
- 操作按钮固定在每行右侧，状态标签彩色。
- 分页器右对齐。

### 8.3 用户端 — 首页
- 顶部问候语 + 进行中工单统计卡片（品牌色背景）。
- 轮播 FAQ 提示条，点击跳转详情。
- 菜单宫格：双列大圆角卡片，图标 + 标题 + 描述。

### 8.4 用户端 — 工单详情
- 状态条居中，背景色随状态变化。
- 信息卡片分段展示，物料账单展示单价、数量、折扣，总金额醒目。
- 操作按钮固定在底部，白色背景顶部分隔线。

### 8.5 聊天页面
- 消息气泡：对方浅灰底 + 边框，己方品牌色底白字。
- 输入栏白色背景，圆角输入框，发送按钮品牌色。
- 新消息自动滚动到底部，提供快速回底按钮。

## 九、响应式与适配

### 9.1 管理端适配
- 主要面向 PC 屏幕（1920px 主流），侧边栏固定，主内容区自适应。
- 当屏幕宽度 < 1200px 时，统计卡片由 6 列变 3 列。
- 表格最小宽度 800px，横向滚动。

### 9.2 小程序适配
- 使用 rpx 单位自适应不同手机屏幕。
- 图片宽度自适应，最大宽度限制 240rpx。
- 底部安全区域适配 iPhone X 以上机型。

---

## 十、设计资源与交付

- **色彩变量**：在 CSS 中定义全局变量，管理端使用 SCSS 变量，小程序端使用 CSS 自定义属性。
- **组件库**：管理端基于 Element Plus 二次定制，小程序端自建组件。
- **图标**：Element Plus Icons 或自制 SVG 图标。
- **字体**：系统默认字体栈，无需额外加载 Web 字体。

---

此规范旨在确保整个后勤维修系统在视觉和交互上高度统一，提升用户体验和开发效率。后续迭代中所有新功能、新页面均应严格遵循本规范。
