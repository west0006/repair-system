## 一、项目概述
### 1.1 项目背景
本项目为线上线下一体化维修管理系统，通过微信小程序为用户提供便捷的报修入口，实现维修工单智能分配、师傅接单处理、物料申领审批、在线支付结算、用户评价反馈全流程数字化管理。系统打通用户端、维修师傅端、后台管理端与物料库存系统，解决传统维修行业派单效率低、流程不透明、结算不规范、库存管理混乱等痛点。

### 1.2 项目目标
- 为用户提供一站式报修服务，支持在线填写报修单、查看工单进度、在线支付、评价反馈
- 实现维修工单按区域智能分配，支持师傅自主接单与拒单，提升派单效率
- 打通物料申领-审批-出库全流程，实现库存数据实时同步
- 集成微信小程序支付，支持在线缴费与退款，保障资金安全
- 提供完整的订单统计、维修记录、评价分析功能，为运营决策提供数据支撑

### 1.3 系统整体架构
系统采用前后端分离架构，分为五个核心模块：
1. **用户端小程序**：用户报修、支付、评价入口
2. **维修师傅端小程序**：工单接收、处理、物料申领入口
3. **系统中台调度系统**：工单分配、支付处理、消息推送、数据统计核心
4. **后台管理系统**：工单管理、用户管理、师傅管理、物料管理、审批管理
5. **物料库存系统**：库存管理、出入库记录、库存预警

### 2.2 用户端流程
1. **统一报修平台入口**
    - 用户通过微信小程序进入统一报修平台，完成微信授权登录
    - 系统自动获取用户基本信息（openid、昵称、头像），无需手动注册

2. **填写报修单**
    - 用户填写报修信息：故障类型、故障描述、地址、联系电话、预约维修时间
    - 支持上传故障图片/视频，辅助维修师傅提前了解故障情况
    - 提交报修单后，系统自动生成唯一工单编号

3. **缴费流程**
    - 系统根据维修类型与预估费用生成缴费提示，推送至用户端
    - 用户通过微信小程序支付完成缴费
    - **缴费成功**：系统标记工单为已缴费，进入派单环节
    - **缴费失败/取消工单**：系统自动结束工单，反馈结果给用户
    - 注：支持维修完成后再缴费的模式，可根据业务需求配置

4. **确认维修与评价**
    - 维修师傅完成维修后，发起确认维修请求
    - 用户在小程序端确认维修完成，核对维修内容与费用
    - 进入用户评价与反馈环节，用户对维修服务进行评分与文字评价
    - 提交评价后，工单正式结束

### 2.3 系统中台调度流程
1. **按区域分配派单**
    - 接收用户提交的报修单后，系统根据报修地址自动匹配对应区域的维修师傅
    - 支持多种派单策略：就近派单、按师傅技能匹配派单、按师傅工作量均衡派单
    - 派单成功后，将工单信息推送至对应维修师傅端

2. **工单状态流转管理**
    - 实时跟踪工单状态：待派单、已派单、已接单、维修中、待确认、已完成、已取消
    - 师傅拒单后，系统自动进行二次分配，将工单重新派发给其他符合条件的师傅
    - 工单超时未接单时，系统自动提醒管理员介入处理

3. **缴费与结算管理**
    - 生成缴费订单，对接微信小程序支付系统
    - 接收支付回调通知，更新订单支付状态
    - 支持退款申请与处理，对接微信退款接口
    - 生成交易账单与资金账单，供财务对账使用

4. **消息推送**
    - 使用**微信小程序一次性订阅消息**向用户与师傅推送关键节点通知
    - 订阅消息模板ID：`IcPrVPItLPl60G03qZq6oAoZV-fP1fyAdOBwZVsGZco`
    - 推送场景：工单派单成功、师傅接单、维修完成、缴费成功、退款到账等
    - 所有开发信息可登录微信公众平台查看与配置

### 2.4 维修师傅端流程
1. **工单接收与处理**
    - 师傅接收系统推送的工单信息，查看报修详情、用户地址、联系电话
    - 确认受理工单，系统将工单标记为已接单，同步通知用户
    - 若无法接单，可发起拒单申请，填写拒单原因，系统自动重新分配

2. **日常巡检与自主报修**
    - 支持师傅发起日常巡检任务，发现故障后自主提交报修单
    - 自主报修单自动进入待派单/待接单列表，由系统或管理员分配处理

3. **物料申领流程**
    - 维修过程中需要使用物料时，师傅在端内发起物料申领申请
    - 填写申领物料名称、数量、规格、用途，提交后台审批
    - 查看申领审批状态：审批中、审批通过、审批驳回
    - 审批驳回时，可修改申领信息后重新提交
    - 审批通过后，物料出库，系统自动同步库存数据

4. **维修完成与确认**
    - 完成维修工作后，师傅在端内填写维修记录：故障原因、维修方案、使用物料明细、实际维修费用
    - 发起确认维修请求，等待用户确认
    - 用户确认后，工单标记为已完成，系统生成维修记录

### 2.5 系统反馈与统计流程
1. **工单列表管理**
    - 实时维护三类工单列表：待派单/待接单列表、已派单/已接单列表、已完成列表
    - 支持按工单状态、时间、区域、师傅等多维度筛选查询

2. **数据记录与统计**
    - 生成维修记录：包含工单信息、维修师傅、维修时间、使用物料、费用明细
    - 记录用户反馈与评价，生成评价统计报表
    - 提供工单量统计、维修成功率统计、用户满意度统计、师傅工作量统计等多维度数据报表

### 2.6 材料库存关联流程
1. **后台审批**
    - 管理员接收师傅提交的物料申领申请，审核申领信息的合理性
    - 核对库存数量，确认是否有足够物料可供出库
    - 审批通过：生成物料出库单，通知仓库管理员出库
    - 审批驳回：填写驳回原因，反馈给申请师傅，支持修改后重新提交

2. **材料出库**
    - 仓库管理员根据审批通过的出库单进行物料出库操作
    - 系统自动扣减对应物料的库存数量，生成出库记录
    - 出库完成后，同步通知申请师傅与系统中台，更新工单物料使用信息

3. **库存管理**
    - 实时维护物料库存数据，支持库存预警功能
    - 当物料库存低于预警值时，自动提醒管理员进行采购
    - 生成库存台账，记录所有物料的出入库明细与当前库存
